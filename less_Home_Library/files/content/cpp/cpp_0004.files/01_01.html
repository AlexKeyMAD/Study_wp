ВСТУПЛЕНИЕ

Цель этой книги - научиться разрабатывать приложения в Windows. Чтобы сделать Итак, я разработал Small Windows, объектно-ориентированную библиотеку классов C ++ для графические приложения в Windows.

Идея состоит в том, чтобы направить вас в программирование для Windows, вводя все больше и больше продвинутые приложения, написанные на C ++ с использованием Small Windows, тем самым скрывая технические подробные сведения об интерфейсе программирования 32-разрядных приложений Windows (Win32 API), который базовая библиотека для разработки под Windows. При таком подходе мы можем сосредоточиться на бизнес-логика без проблем с основными техническими деталями. Если ты заинтересован зная, как работает Win32 API, вторая часть этой книги дает подробное описание того, как реализована Small Windows.

Эта книга состоит из двух частей, первая из которых описывает разработанные приложения. в C ++ с Small Windows. Хотя в некоторых книгах есть много примеров, эта книга только включает шесть примеров, среди которых последние четыре довольно продвинутые: игра Тетрис, программа для рисования, текстовый процессор и программа для работы с электронными таблицами. Обратите внимание, что эта книга не только учебник по программированию для Windows, но также и руководство по разработке объектно-ориентированные графические приложения.

Вторая часть содержит подробное описание реализации Small Windows в API Win32. Обратите внимание, что Win32 API не будет представлен до второй части. Некоторые из вы можете быть удовлетворены высокоуровневыми аспектами Small Windows и захотите только изучить проблемы, связанные с конкретным приложением, в то время как другие могут захотеть прочитать вторую часть, чтобы понять, как классы, методы и макросы Small Windows реализованы в API Win32.

Естественно, мне известно о существовании современных объектно-ориентированных библиотек классов для Windows. Однако цель этих библиотек - облегчить разработчикам задачу: скрытие деталей архитектуры, что также мешает разработчику использовать Архитектура Windows в полной мере. Несмотря на то, что Win32 API существует уже давно в то время как я считаю это лучшим способом разработки профессиональных приложений для Windows и разбираться в архитектуре Windows.
